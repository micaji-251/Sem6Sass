SASS

Descripcion

Un preprocesador es un lenguaje que parte de CSS (stylus, section), al compilarse se vuelve CSS, section no reconoce el navegador (solo reconoce HTML, CSS y JAVASCRIPT).

Desventajas para CSS plano

Escalabilidad: puede generar problemas de lectura

Ventajas de sass

Nos permite la modularisación y tiene gran posibilidad de soporte por su comunidad.

  • Uso de variables

    Se nombran variables haciendo uso de un signo de $ dinero, este se coloca adelante de la variable

    $first-color:red;

  • Anidamiento

    Permite que en el codigo SCSS se pueda colocar una etiqueta dentro de otra

    .container{ width:300px; a{text_decoration:none;}}

  • Nesting o Modularización

    Permite que se divida el codigo SCSS en varios archivos parciales, estos se nombran con un _ adelante para que no generen un archivo de copilación independiente, se tiene que ir importando

    _menu.scss, se coloca en el principal @import "./module/menu.scss" para que funcione

Nomenclatura de CSS

Uso de BEM (Bloque, Elemento, Modifier)

B - Bloque: es el nombre de la section que lo contiene

E - Elemento: es el nombre del elemento asociado que es parte, item (li), link(a), entre otros.

M - Modifier: se utiliza si el elemento tiene un cambio de estado a partir de una acción, esto se maneja con JAVASCRIPT

Arquitectura de CSS

La arquitectura en CSS sigue usualmente la forma SMACSS, esta separa los estilos CSS en distintas carpetas:

  1. Base

    esta carpeta contiene lo que son valores de reseteo para los distintos elementos, de esta forma de no tienen valor por defecto, reemplaza lo que es el *, *::after *::before{ box-sizing:box-border; padding:0; margin:0;}

    necolas.github.io/normalize.css/8.0.1/normalize.css

  2. Layout

    en esta sección va todo lo que son contenedores. Iria especificaciones de estos no relacionados a font, sino a width o heigh, padding, margin. Para colocar grid a un componente.

    .container{     max-width: 200px; }

  3. module

    en este apartado va todo los componentes de una pagina, las secciónes: header, footer, menu de navegación, cada una en una pagina scss

  4. state

    en esta carpeta va todo los elementos que varian de estado con JAVASCRIPT a partir de alguna interacción. En el ejemplo colocamos lo que era un color diferente en el menu de navegación para la sección en la que estabamos.

  5. theme

    va todo lo que sont font y colores

    a{font-family:arial,helvetica,sans-serif;}
